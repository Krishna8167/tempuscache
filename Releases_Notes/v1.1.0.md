### In-Memory TTL Cache

-   Backed by `map[string]Item`

-   Each `Item` stores:

    -   `Value` (any type)

    -   `ExpiryTime` (precise expiration timestamp)

-   Common TTL duration configured at initialization

* * * * *

### Concurrency Safety (`sync.RWMutex`)

-   Concurrent reads via `RLock`

-   Exclusive writes for mutation (`Set`, eviction)

-   Verified using `go test -race`

Designed for:

-   Web servers

-   Background workers

-   Microservices

-   Concurrent pipelines

* * * * *

### Hybrid Expiration Strategy

#### Lazy Expiration

-   Every `Get` checks expiration

-   Expired keys are removed immediately

-   Stale data is never returned

#### Active Background Eviction

-   Background goroutine periodically scans the cache

-   Removes expired entries

-   Prevents memory retention of stale keys
